# Quick Start

## Purpose

Provide a structured starting point for conducting an investigation using this workflow.

## When to Use

Use when beginning any investigation involving potential compromise, persistence, or suspicious system behavior.

## Workflow Entry

All investigations begin at:

phases/phase-00-initial-triage

## Investigation Flow

1. Perform initial triage to determine if indicators exist
2. If indicators are present, immediately transition to preservation
3. Capture baseline system state before deep inspection
4. Run suspicion sweeps to identify persistence and network anomalies
5. Preserve all relevant artifacts before modification or removal
6. Perform deep analysis and correlate findings
7. Validate results through re-checks and monitoring
8. Document findings and produce a report

## Critical Rules

* Do not modify the system before preservation steps are completed
* Do not trust initial indicators without validation
* Do not rely on a single script output
* Always assume evidence may be lost if mishandled

## Notes

This workflow is designed to support defensible investigations. Every action should be explainable, repeatable, and documented.
